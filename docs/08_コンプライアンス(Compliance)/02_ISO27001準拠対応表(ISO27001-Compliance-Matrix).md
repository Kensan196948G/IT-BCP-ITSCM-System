## 1. 本文書の目的

本文書は、IT-BCP-ITSCMシステムがISO/IEC 27001:2022 附属書A（Annex A）の管理策に対してどのように準拠しているかを体系的にマッピングしたものである。特に事業継続管理およびICTの準備に関連する管理策（A.5.29、A.5.30）を中心に、関連する情報セキュリティ管理策への対応状況を詳細に記載する。

## 2. 適用範囲

- 対象組織: みらい建設工業株式会社
- 対象システム: IT事業継続管理システム（IT-BCP-ITSCM-System）
- 対象規格: ISO/IEC 27001:2022 Annex A（特にA.5.29、A.5.30および関連管理策）

## 3. 準拠ステータスの定義

| ステータス | 説明 |
|-----------|------|
| 対応済 | 本システムにて機能実装済みであり、エビデンスが自動生成される |
| 一部対応済 | 機能の一部が実装済みであり、追加の運用手順で補完する |
| 対応予定 | 今後の開発計画に含まれている |
| 対象外 | 本システムの適用範囲外（組織的対応が必要） |
| 間接対応 | 本システムの機能が間接的に当該管理策の実現を支援する |

## 4. 主要管理策の詳細対応表

### 4.1 A.5.29 事業の中断・阻害時の情報セキュリティ ★主要管理策★

#### 管理策の概要

| 項目 | 内容 |
|------|------|
| 管理策番号 | A.5.29 |
| 管理策名称 | 事業の中断・阻害時の情報セキュリティ（Information security during disruption） |
| 管理策の内容 | 組織は、事業の中断・阻害時に情報セキュリティを適切なレベルに維持するための計画を策定しなければならない |
| カテゴリ | 組織的管理策（Organizational controls） |

#### 本システムの対応詳細

##### (1) 情報セキュリティ継続計画の策定

| 対応項目 | 実装機能 | 詳細説明 |
|---------|---------|---------|
| セキュリティ要件の特定 | BIA機能 | 事業中断時に維持すべき情報セキュリティレベルをBIAで特定。各業務プロセスに紐づくセキュリティ要件（機密性・完全性・可用性）を定義・管理 |
| セキュリティ継続計画 | ITSCM計画管理 | 復旧手順書内にセキュリティ維持手順を組み込み。フェイルオーバー時のアクセス制御、暗号化要件、認証要件を計画に明記 |
| セキュリティリスク評価 | リスク評価機能 | 中断時に発生し得るセキュリティリスク（不正アクセス、データ漏洩、通信傍受）を評価。代替手段使用時のセキュリティリスクも考慮 |
| セキュリティ対策の代替手段 | 代替手段管理 | 通常のセキュリティ対策が機能しない場合の代替策を定義。VPN障害時の代替暗号化通信、認証サーバ障害時の代替認証手段等 |

##### (2) セキュリティ継続のテスト・評価

| 対応項目 | 実装機能 | 詳細説明 |
|---------|---------|---------|
| セキュリティ要件の訓練シナリオ | 訓練シナリオ管理 | サイバー攻撃シナリオ（ランサムウェア、DDoS、標的型攻撃）のテンプレートを提供。セキュリティインシデント発生時の復旧手順を検証 |
| セキュリティレベル維持の検証 | 訓練結果記録 | 訓練中のセキュリティ対策の有効性を評価。アクセス制御の維持状況、データ保護状況を記録・検証 |
| セキュリティ改善提案 | 改善提案機能 | 訓練結果からセキュリティ関連の改善点を自動抽出。次回訓練・計画改訂へのフィードバック |

##### (3) エビデンス

| エビデンス種別 | 生成方法 | 保管期間 |
|---------------|---------|---------|
| セキュリティ継続計画書 | 計画管理機能からPDF出力 | 5年 |
| セキュリティリスク評価結果 | BIA機能からレポート生成 | 5年 |
| セキュリティ訓練結果レポート | 訓練管理機能から自動生成 | 5年 |
| セキュリティ改善提案記録 | 改善管理機能から出力 | 3年 |

| 準拠ステータス | **対応済** |
|---------------|-----------|

### 4.2 A.5.30 事業継続のためのICTの準備 ★主要管理策★

#### 管理策の概要

| 項目 | 内容 |
|------|------|
| 管理策番号 | A.5.30 |
| 管理策名称 | 事業継続のためのICTの準備（ICT readiness for business continuity） |
| 管理策の内容 | ICTの準備は、事業継続の目的及びICT継続の要求事項に基づいて、計画し、実施し、維持し、テストしなければならない |
| カテゴリ | 組織的管理策（Organizational controls） |

#### 本システムの対応詳細

##### (1) ICT継続要件の特定

| 対応項目 | 実装機能 | 詳細説明 |
|---------|---------|---------|
| BIAに基づくICT要件特定 | BIA機能 | 業務プロセスに必要なICTサービスの特定、各ICTサービスのRTO/RPO要件の決定、ICTサービス間の依存関係分析 |
| ICTリスクアセスメント | リスク評価機能 | ICTインフラの脆弱性評価、単一障害点（SPOF）の特定、建設業特有のICTリスク（現場通信環境、モバイルデバイス依存等）の評価 |
| ICTサービスカタログ | サービス管理機能 | 対象ICTサービスの一覧管理、各サービスの重要度分類、サービス提供者（内部/外部）の管理 |

##### (2) ICT継続計画の策定

| 対応項目 | 実装機能 | 詳細説明 |
|---------|---------|---------|
| ICT復旧手順書 | 復旧手順書管理 | システム別・サービス別の詳細復旧手順、復旧前提条件（電源、ネットワーク、要員）の明記、復旧確認手順（チェックリスト形式） |
| RTO/RPO管理 | RTO/RPO管理機能 | サービス別RTO/RPO目標値の設定・管理、RTO/RPO目標と実績の差分分析、目標未達時のアラート・エスカレーション |
| 代替手段計画 | 代替手段管理 | DRサイト（災害復旧サイト）の管理、クラウドベースのフェイルオーバー設定、代替通信手段（衛星回線等）の管理、建設現場向けオフライン対応手順 |
| 優先復旧順序 | 優先復旧順序管理 | BIA結果に基づく復旧優先度の自動算出、サービス依存関係を考慮した復旧順序の最適化、優先復旧順序表の自動生成 |
| ベンダー連携 | ベンダー管理機能 | 復旧支援ベンダーの連絡先・契約情報一元管理、ベンダーSLA（復旧対応時間、オンサイト対応時間）の管理、ベンダーのBCP対応状況評価・監査 |

##### (3) ICT継続のテスト

| 対応項目 | 実装機能 | 詳細説明 |
|---------|---------|---------|
| テスト計画策定 | 訓練管理機能 | ICT継続テストの年間スケジュール管理、テスト種別（テーブルトップ/技術テスト/統合テスト）の選択・管理 |
| テスト実施 | 訓練実行機能 | テーブルトップ演習：ICT復旧シナリオに基づくディスカッション形式の演習。技術テスト：フェイルオーバーテスト、バックアップリストアテスト。統合テスト：業務部門を含む統合的な復旧訓練。各テストのタイムトラッキング（RTO計測） |
| テスト結果分析 | 訓練結果分析機能 | RTO達成/未達の自動判定、復旧手順の有効性評価、ボトルネックの特定・改善提案 |
| テストレポート | レポート自動生成 | テスト結果の包括的レポート（サマリー、詳細結果、改善提案）をPDF形式で自動生成 |

##### (4) ICT継続の維持・改善

| 対応項目 | 実装機能 | 詳細説明 |
|---------|---------|---------|
| 定期的な計画見直し | 見直しスケジュール管理 | ICT環境変更時の計画見直しトリガー、定期見直しリマインダー（四半期/年次） |
| 改善管理 | 改善提案管理機能 | テスト結果からの改善提案自動生成、インシデント対応からの教訓反映、改善タスクの進捗管理 |
| KPI管理 | ダッシュボード機能 | ICT継続成熟度の評価、RTO/RPO達成率の傾向分析、訓練実施率・改善実施率のトラッキング |

みらい建設工業の建設現場特有のICTリスクに対する本システムの対応を以下に記述する。
