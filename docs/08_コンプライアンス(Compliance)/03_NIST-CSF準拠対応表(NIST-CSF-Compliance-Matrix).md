## 1. 本文書の目的

本文書は、IT-BCP-ITSCMシステムがNIST Cybersecurity Framework（CSF）2.0に対してどのように準拠しているかを体系的にマッピングしたものである。本システムの中核機能であるRECOVER（RC）ファンクションを中心に、関連するGOVERN（GV）、IDENTIFY（ID）、PROTECT（PR）、DETECT（DE）、RESPOND（RS）ファンクションとの関連も含めて詳細に記載する。

## 2. 適用範囲

- 対象組織: みらい建設工業株式会社
- 対象システム: IT事業継続管理システム（IT-BCP-ITSCM-System）
- 対象規格: NIST Cybersecurity Framework 2.0（特にRECOVERファンクション）

## 3. NIST CSF 2.0 ファンクション概要

| ファンクション | コード | 説明 | 本システムとの関連度 |
|--------------|--------|------|-------------------|
| GOVERN | GV | サイバーセキュリティリスク管理のガバナンス | 中 |
| IDENTIFY | ID | 組織のサイバーセキュリティリスクの理解 | 高 |
| PROTECT | PR | サイバーセキュリティリスクを管理するためのセーフガード | 中 |
| DETECT | DE | サイバーセキュリティ攻撃・侵害の検知 | 低 |
| RESPOND | RS | サイバーセキュリティインシデントへの対応 | 高 |
| RECOVER | RC | サイバーセキュリティインシデントからの復旧 | **最高（中核機能）** |

## 4. RECOVER（RC）ファンクション - 詳細対応表 ★中核対応領域★

### 4.1 RC.RP: 復旧計画（Recovery Planning）

#### RC.RP-01: インシデント復旧計画の実行

| 項目 | 内容 |
|------|------|
| サブカテゴリID | RC.RP-01 |
| サブカテゴリ | インシデント復旧計画が実行される（The recovery portion of the incident response plan is executed once initiated from the incident response process） |
| 要求の詳細 | サイバーセキュリティインシデント発生後、定められた復旧計画に従って復旧活動を開始し、計画に沿って実行すること |
| 本システムの対応機能 | |

##### 対応機能の詳細

| 機能名 | 対応内容 |
|--------|---------|
| **緊急対応指揮機能** | インシデント発生検知後、定義済みの指揮系統を自動活性化。指揮者・対応メンバーへの自動通知、緊急対応体制の即座確立。建設現場を含む全拠点への同時通知対応 |
| **復旧手順実行管理** | 復旧手順書に基づくステップバイステップの実行ガイド。各ステップの実行者割当、実行状況のリアルタイム追跡。手順の完了チェックリスト、前提条件の自動確認 |
| **RTOダッシュボード** | 復旧開始からの経過時間をリアルタイム表示。各サービスのRTO目標と実績の差分を可視化。RTO超過アラート、エスカレーション自動実行 |
| **タスク管理** | 復旧タスクの担当者割当・優先度設定・進捗追跡。タスク間の依存関係管理、並行実行可能タスクの特定。モバイル対応による現場からのタスク更新 |
| **優先復旧順序管理** | BIA結果に基づく復旧優先度の自動算出。サービス依存関係を考慮した最適復旧順序の提示。建設業の安全管理システム等の最優先復旧対象の明示 |

| 準拠ステータス | **対応済** |
|---------------|-----------|
| エビデンス | 復旧実行記録、RTOダッシュボードログ、タスク管理ログ、復旧完了報告書 |
| エビデンス生成方法 | インシデント対応中は自動記録。対応完了後「インシデント > 対応レポート生成」でPDF出力 |

### 4.2 RC.CO: 復旧コミュニケーション（Recovery Communication）

#### RC.CO-01: 復旧活動に関する内部コミュニケーション

| 項目 | 内容 |
|------|------|
| サブカテゴリID | RC.CO-01 |
| サブカテゴリ | 復旧活動及び正常な運用の復旧に関して、権限のある内部及び外部の利害関係者に通知される |
| 要求の詳細 | 復旧活動の状況および正常運用への復旧進捗について、適切な利害関係者にタイムリーに通知すること |
| 本システムの対応機能 | |

##### 対応機能の詳細

| 機能名 | 対応内容 |
|--------|---------|
| **状況報告自動化** | 復旧進捗の定期自動報告（設定可能な間隔：15分/30分/1時間）。経営層向けサマリーレポート、技術チーム向け詳細レポートの自動生成・配信。RTOダッシュボードのリアルタイム共有 |
| **連絡網管理** | 内部利害関係者（経営層、IT部門、事業部門、建設現場管理者）の連絡先一元管理。連絡手段の多重化（メール、SMS、電話、チャット、社内ポータル）。連絡到達確認機能 |
| **外部連絡管理** | 顧客・取引先への状況通知テンプレート。規制当局（国土交通省等）への報告手順・テンプレート。ベンダー・協力会社への連絡管理 |
| **エスカレーション** | 復旧遅延時の自動エスカレーション。RTO超過予測時の早期警告。段階的エスカレーションルールの定義・実行 |

| 準拠ステータス | **対応済** |
|---------------|-----------|
| エビデンス | 状況報告送信記録、連絡実施記録、外部通知記録、エスカレーション実行記録 |
| エビデンス生成方法 | 全通知の送受信記録が自動保存。「インシデント > コミュニケーション記録」からPDF出力 |

### 4.3 RC.IM: 復旧の改善（Recovery Improvements）

#### RC.IM-01: 復旧計画・プロセスへのインシデント教訓の反映

| 項目 | 内容 |
|------|------|
| サブカテゴリID | RC.IM-01 |
| サブカテゴリ | インシデントからの教訓が復旧計画とプロセスに反映される |
| 要求の詳細 | 実際のインシデント対応および復旧活動から得られた教訓を体系的に収集し、復旧計画とプロセスの改善に反映すること |
| 本システムの対応機能 | |

##### 対応機能の詳細

| 機能名 | 対応内容 |
|--------|---------|
| **事後レビュー（ポストモーテム）** | インシデント対応完了後の包括的事後レビュー機能。時系列での対応活動の振り返り、各対応アクションの有効性評価、RTO達成/未達の原因分析。建設現場特有の課題（通信環境、要員配置等）の分析 |
| **教訓データベース** | 教訓の体系的な登録・検索機能。インシデント種別、影響範囲、対応結果によるカテゴリ分類。類似インシデントとの関連付け。過去の教訓の訓練シナリオへの自動組み込み提案 |
| **改善提案自動生成** | 事後レビュー結果からの改善提案自動生成。改善提案の優先度判定（RTO影響度、再発可能性等に基づく）。改善提案の計画への反映ワークフロー |
| **改善タスク管理** | 改善提案のタスク化（担当者割当、期限設定）。改善タスクの進捗追跡。改善実施後の有効性評価 |
| **計画自動更新提案** | 教訓に基づく復旧計画の更新提案。RTO/RPO目標値の見直し提案。復旧手順の改訂提案 |

| 準拠ステータス | **対応済** |
|---------------|-----------|
| エビデンス | 事後レビューレポート、教訓データベース記録、改善提案一覧、改善タスク進捗記録、計画更新履歴 |
| エビデンス生成方法 | インシデント収束後に「インシデント > 事後レビュー」から実行。教訓は自動的にデータベースに登録され、「改善管理 > 教訓一覧」から検索・出力可能 |
