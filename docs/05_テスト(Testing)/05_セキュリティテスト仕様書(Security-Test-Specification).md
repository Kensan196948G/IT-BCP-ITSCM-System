## 1. 概要

### 1.1 目的

本文書は、IT事業継続管理システムのセキュリティテスト仕様を定義する。本システムは事業継続に直結する重要な情報を取り扱うため、ISO 27001、NIST CSF に準拠したセキュリティ水準が求められる。OWASP Top 10 をはじめとする既知の脆弱性への対策を検証し、認証・認可・暗号化・入力検証等の各セキュリティ機能が正しく実装されていることを確認する。

### 1.2 準拠規格・基準

| 規格・基準 | 適用範囲 |
|-----------|---------|
| ISO 27001:2022 | 情報セキュリティマネジメントシステム全般 |
| ISO 20000-1:2018 | ITサービスマネジメント |
| NIST CSF 2.0 | サイバーセキュリティフレームワーク |
| OWASP Top 10 (2021) | Webアプリケーション脆弱性対策 |
| OWASP ASVS 4.0 | アプリケーションセキュリティ検証標準 |
| CWE/SANS Top 25 | ソフトウェア脆弱性 |

### 1.3 セキュリティテストの分類

| テスト種別 | 説明 | 頻度 | ツール |
|-----------|------|------|-------|
| SAST（静的解析） | ソースコードの静的セキュリティ解析 | PR毎（CI） | CodeQL, Bandit, ESLint Security |
| DAST（動的解析） | 実行中アプリケーションの脆弱性スキャン | リリース前 | OWASP ZAP |
| SCA（ソフトウェア構成分析） | 依存パッケージの脆弱性チェック | 毎日（CI） | pip-audit, npm audit, Trivy |
| 認証テスト | 認証機能のセキュリティ検証 | リリース前 | 手動 + 自動 |
| 認可テスト | アクセス制御の検証 | リリース前 | 手動 + 自動 |
| 暗号化テスト | データ暗号化の検証 | リリース前 | 手動 + SSL Labs |
| ペネトレーションテスト | 擬似的な攻撃による脆弱性検証 | 四半期 | 外部ベンダー |
| シークレットスキャン | コード内の機密情報検出 | PR毎（CI） | Gitleaks |

## 6. ペネトレーションテスト計画

### 6.1 概要

| 項目 | 内容 |
|------|------|
| テスト種別 | ブラックボックス + グレーボックス |
| 実施者 | 外部セキュリティベンダー |
| 頻度 | 四半期ごと |
| 期間 | 5営業日 |
| 対象環境 | staging環境（本番同等構成） |
| 報告形式 | エグゼクティブサマリー + 技術的詳細レポート |

### 6.2 テスト範囲

| カテゴリ | 対象 | テスト内容 |
|---------|------|-----------|
| Webアプリケーション | フロントエンド（Next.js） | XSS, CSRF, クリックジャッキング, セッション管理 |
| API | バックエンド（FastAPI） | インジェクション, 認証・認可バイパス, IDOR, レート制限 |
| 認証 | Azure AD + JWT | 認証フローの脆弱性, トークン管理 |
| インフラ | Azure Container Apps | コンテナエスケープ, ネットワークセグメンテーション |
| データベース | PostgreSQL | SQLインジェクション, アクセス制御 |
| WebSocket | RTOダッシュボード | WebSocket固有の攻撃, メッセージ改ざん |

### 6.3 除外事項

- DoS/DDoS攻撃テスト（別途実施）
- 物理的セキュリティテスト
- ソーシャルエンジニアリング
- Azure プラットフォーム自体への攻撃

### 6.4 実施スケジュール

| 日程 | 内容 |
|------|------|
| Day 1 | スコープ確認、情報収集、自動スキャン実行 |
| Day 2-3 | 手動テスト（認証・認可、インジェクション、ビジネスロジック） |
| Day 4 | WebSocket、API深堀りテスト、再テスト |
| Day 5 | レポート作成、報告会 |

### 6.5 脆弱性分類と対応SLA

| 重要度 | CVSS スコア | 対応SLA | 例 |
|--------|-----------|--------|-----|
| Critical | 9.0 - 10.0 | 即時対応（24時間以内） | リモートコード実行、認証バイパス |
| High | 7.0 - 8.9 | 3営業日以内 | SQLインジェクション、権限昇格 |
| Medium | 4.0 - 6.9 | 次スプリント内 | XSS、情報漏洩 |
| Low | 0.1 - 3.9 | 計画的に対応 | 情報開示、ベストプラクティス違反 |
| Informational | 0 | 対応任意 | 改善提案 |
