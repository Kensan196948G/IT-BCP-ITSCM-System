## 1. 概要

本文書は、IT事業継続管理システムの各機能に対する詳細なテストケースを定義する。テストケースは機能カテゴリごとに分類し、テストID、カテゴリ、前提条件、手順、期待結果、優先度を明記する。

### 1.1 テストケースID体系

```
TC-<カテゴリ>-<連番>
```

| カテゴリコード | 機能 |
|-------------|------|
| BCP | BCP計画管理 |
| DRL | 訓練管理 |
| INC | インシデント対応 |
| DSH | ダッシュボード |
| BIA | ビジネスインパクト分析 |
| AUT | 認証・認可 |
| NTF | 通知 |
| PWA | PWA / オフライン |
| API | API共通 |

### 1.2 優先度定義

| 優先度 | 定義 |
|--------|------|
| P1 (Critical) | リリースブロッカー。必ず実行・合格が必要 |
| P2 (High) | 主要機能のテスト。合格が強く推奨 |
| P3 (Medium) | 補助機能のテスト。時間があれば実行 |
| P4 (Low) | エッジケース、改善系テスト |
