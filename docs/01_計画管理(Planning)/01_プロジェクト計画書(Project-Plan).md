## 1. プロジェクト概要

### 1.1 プロジェクト名称

**IT事業継続管理システム（IT-BCP-ITSCM-System）**

### 1.2 プロジェクト概要

本プロジェクトは、災害およびサイバー攻撃発生時におけるIT復旧計画の策定・管理、BCP訓練の実施・記録、RTO（Recovery Time Objective）ダッシュボードによるリアルタイム監視を統合的に提供するプラットフォームを構築するものである。

ISO20000 ITSCMプロセスに完全準拠し、組織のIT事業継続管理能力を抜本的に強化する。

### 1.3 背景と課題

| # | 現状の課題 | 影響 |
|---|-----------|------|
| 1 | IT復旧計画がExcel・Wordで個別管理されており、一元的な管理ができていない | 災害時に最新の計画を迅速に参照できない |
| 2 | BCP訓練の実施記録・評価が属人的で標準化されていない | 訓練の効果測定・改善サイクルが機能していない |
| 3 | RTO/RPOの達成状況をリアルタイムで把握する手段がない | 復旧目標の未達に気付くのが遅れる |
| 4 | ISO20000 ITSCM要求事項への準拠状況が可視化されていない | 審査対応に多大な工数がかかる |
| 5 | 災害時のオフライン環境での計画参照ができない | ネットワーク障害時に復旧手順を確認できない |

---

## 2. プロジェクト目的

### 2.1 ビジネス目的

1. **IT事業継続管理の体系化**: ISO20000 ITSCM要求事項に完全準拠した管理プロセスの実装
2. **復旧時間の短縮**: RTO達成率を現状の70%から95%以上に向上
3. **訓練品質の向上**: BCP訓練の計画・実施・評価・改善サイクルの自動化
4. **経営可視性の向上**: RTOダッシュボードによるリアルタイム状況把握の実現
5. **レジリエンス強化**: サイバー攻撃を含むあらゆるIT障害への対応力強化

### 2.2 技術目的

1. モダンアーキテクチャによる拡張性・保守性の高いシステム構築
2. PWA対応によるオフライン環境での計画参照機能の実現
3. マルチリージョン構成によるシステム自体の高可用性確保
4. API駆動設計による外部システム連携の容易性確保

## 3. スコープ

### 3.1 スコープ内（In Scope）

| カテゴリ | 機能・作業範囲 |
|---------|--------------|
| BCP計画管理 | IT復旧計画の作成・承認ワークフロー・版管理・配布管理 |
| RTO/RPO管理 | 目標値設定・実績収集・達成率計算・トレンド分析 |
| 訓練管理 | 訓練シナリオ作成・スケジュール管理・実施記録・評価・改善提案 |
| RTOダッシュボード | リアルタイムRTO/RPO表示・アラート・レポート自動生成 |
| コンプライアンス管理 | ISO20000 ITSCM・ISO27001・NIST CSF準拠状況管理 |
| 通知・アラート | メール/Teams/Slack連携・エスカレーション |
| PWAオフライン | 復旧計画・手順書のオフライン参照 |
| マルチリージョンDR | Azure Container Apps 東日本/西日本リージョン冗長構成 |
| 認証・認可 | Azure AD連携・RBAC・多要素認証 |

### 3.2 スコープ外（Out of Scope）

| # | 除外項目 | 理由 |
|---|---------|------|
| 1 | 物理的な事業継続計画（非IT） | 本システムはIT-BCPに特化 |
| 2 | 既存監視ツール（Zabbix等）の置き換え | 連携のみ対応 |
| 3 | ネットワーク機器の自動復旧制御 | 将来フェーズで検討 |
| 4 | 海外リージョンへの展開 | 国内リージョンを優先 |
| 5 | レガシーシステムのリプレース | 本プロジェクトの範囲外 |

## 6. スケジュール概要

### 6.1 全体スケジュール

| フェーズ | 期間 | 開始 | 終了 |
|---------|------|------|------|
| Phase 1: 基盤構築 | 2ヶ月 | 2026年4月 | 2026年5月 |
| Phase 2: コア機能開発 | 3ヶ月 | 2026年6月 | 2026年8月 |
| Phase 3: 高度機能開発 | 2ヶ月 | 2026年9月 | 2026年10月 |
| Phase 4: DR・本番展開 | 1ヶ月 | 2026年11月 | 2026年11月 |

**プロジェクト期間合計: 8ヶ月（2026年4月 - 2026年11月）**

### 6.2 フェーズ概要

**Phase 1: 基盤構築（2ヶ月）**
- 開発環境・CI/CDパイプライン構築
- Azure基盤インフラ構築（IaC）
- 認証・認可基盤（Azure AD連携）
- 基本UIフレームワーク・共通コンポーネント
- データベース設計・マイグレーション基盤

**Phase 2: コア機能開発（3ヶ月）**
- BCP計画管理機能（CRUD・承認ワークフロー・版管理）
- RTO/RPO管理機能（目標設定・実績入力・達成率計算）
- 訓練管理機能（シナリオ・スケジュール・実施記録・評価）
- コンプライアンス管理機能（ISO20000 ITSCMチェックリスト）

**Phase 3: 高度機能開発（2ヶ月）**
- RTOダッシュボード（リアルタイム表示・チャート・アラート）
- PWAオフライン機能（Service Worker・キャッシュ戦略）
- 通知・アラート機能（メール・Teams・Slack連携）
- レポート自動生成機能

**Phase 4: DR・本番展開（1ヶ月）**
- マルチリージョン構成構築（西日本リージョン）
- DR検証・フェイルオーバーテスト
- 性能テスト・セキュリティテスト
- 本番デプロイ・運用移行
