🔐 ゼロトラスト ID統合ガバナンスシステム
EntraID Connect × HENGEONE × AD 統合アイデンティティ管理プラットフォーム
📁 プロジェクト名：ZeroTrust-ID-Governance 🔗 リポジトリ名：Kensan196948G/ZeroTrust-ID-Governance
🆕 新規開発 🔴 優先度：高 ISO27001 A.5.15〜A.8.2 NIST PROTECT PR.AA
🎯 システム概要

既存のEntraID Connect・HENGEONE（SSO/MFA）・ADを統合し、600名分のユーザーライフサイクル（入社〜異動〜退職）を自動管理。建設業特有の「現場勤務者・協力会社・季節社員」の一時的アクセス制御をゼロトラスト原則で実装。IT部門7名の手動運用工数を大幅に削減する。

💡 開発背景・課題
  • AD・EntraID・HENGEONEが個別管理で入退社時の棚卸しが手動
  • 協力会社社員・季節社員への一時アクセス付与が管理簿ベースで証跡不十分
  • ISO27001 A.9（アクセス制御）・NIST CSF PROTECT の要求事項に不整合
  • IT部門7名で600名分のアカウント管理を手動運用中、工数圧迫が深刻
📌 機能要件
  • 【FR-001】AD/EntraID統合ユーザーライフサイクル管理：入社・異動・退職の自動プロビジョニング/デプロビジョニング
  • 【FR-002】HENGEONE SSO統合ポリシー管理：アプリ別アクセス権限マトリックスの可視化・承認ワークフロー
  • 【FR-003】協力会社・一時アカウント管理：期限付き発行・自動失効・上長承認フロー
  • 【FR-004】特権アカウント管理（PAM）：管理者権限の使用申請・タイムボックス承認・セッション録画
  • 【FR-005】ID棚卸し自動化：四半期ごとの全ユーザー権限レビュー・不要権限自動検出・承認通知
  • 【FR-006】MFA適用状況ダッシュボード：未設定ユーザーの自動検出・督促通知
  • 【FR-007】監査ログ統合：AD・EntraID・HENGEONEのログ統合・不正アクセス検知アラート
  • 【FR-008】ISO27001 A.9準拠レポート自動生成：アクセス制御台帳・棚卸し証跡のPDF出力
📌 非機能要件
  • 【NFR-001】可用性：99.9%以上（IT基盤システムのため高可用性必須）
  • 【NFR-002】レスポンス：ユーザー検索・権限変更操作3秒以内
  • 【NFR-003】監査ログ保存：5年間保存・改ざん防止（ハッシュチェーン）
  • 【NFR-004】RBAC：Admin/Operator/Auditor/ReadOnly の4段階権限制御

[ユーザー/管理者] → HTTPS + EntraID SSO → [React フロントエンド] ↓ REST API [FastAPI バックエンド] ├── Microsoft Graph API → EntraID / M365 ├── LDAP/AD Client → オンプレAD (外部DC) ├── HENGEONE REST API → SSO/MFA管理 ├── ワークフローエンジン → 承認/PAM ├── 棚卸しスケジューラー → 四半期自動実行 └── レポートエンジン → ISO27001 A.9準拠PDF ↓ Webhook [Microsoft Teams / Outlook 通知]

🎯 システム概要

建設業特有の分散IT環境（本社・支店・現場事務所・外部DC）のセキュリティログを一元集約するSIEM基盤。AutoCAD・Revit（BIM）・現場IoT機器からのログを収集し、ランサムウェア・不正アクセス・内部不正を早期検知するプラットフォーム。

💡 開発背景・課題
  • 建設業界はランサムウェア被害増加中。設計図・施工データが標的に
  • 現場PCがサプライチェーン攻撃の入口になりやすい（USBウイルス等）
  • NIST CSF 2.0 DETECT（DE.CM）の継続的セキュリティ監視が未実装
  • ファイルサーバー（外部DC）への大量アクセス等の異常検知手段がない
📌 機能要件
  • 【FR-001】ログ集約：AD/EntraID・ファイルサーバー・Exchange Online・M365セキュリティログの一元収集
  • 【FR-002】リアルタイム異常検知：不審ログイン・大量ファイルアクセス・権限昇格試行の即時アラート
  • 【FR-003】ランサムウェア早期検知：ファイルサーバーの短時間大量暗号化操作をパターン検知
  • 【FR-004】現場IoT機器監視：施工現場のネットワーク機器・カメラ・センサーのログ収集と不審通信検知
  • 【FR-005】インシデント自動起票：ITSM-SystemのAPIと連携し検知イベントを自動チケット化
  • 【FR-006】NIST CSF 2.0 6機能別セキュリティKPIダッシュボード可視化
  • 【FR-007】ISO27001監査証跡：セキュリティイベントの証跡保存・コンプライアンスレポート出力
  • 【FR-008】M365 Defender連携：Microsoft 365 Defenderのアラートとの統合・重複排除

📋 建設業 統合リスク＆コンプライアンス管理システム（GRC）
ISO27001・NIST CSF 2.0・建設業法・公共工事品確法 多法令準拠の統合GRC基盤
📁 プロジェクト名：Construction-GRC-System 🔗 リポジトリ名：Kensan196948G/Construction-GRC-System
🆕 新規開発 🔴 優先度：高 ISO27001 全93管理策 NIST GOVERN GV.RM
🎯 システム概要

建設・土木業界固有のリスク（施工不良・安全事故・労務問題）とITセキュリティリスクを統合管理するGRCプラットフォーム。ISO27001の情報資産リスクアセスメントと、建設業法・公共工事品確法・労働安全衛生法のコンプライアンス管理を一元化する。

💡 開発背景・課題
  • ISO27001認証取得・維持のためのリスクアセスメントが手動（Excelベース）
  • 建設業法改正・公共工事品確法への対応状況が各部門バラバラに管理
  • 現場安全点検・労災記録・是正措置の追跡に統合ツールがない
  • NIST CSF 2.0 GOVERN（GV.RM）のリスク管理フレームワーク未整備
📌 機能要件
